S1
Stage 1 — 用 AI
跨越第一跳：從「用 AI 聊天」到「用 AI 工作」
M1-1 AI 聊天做作品

第一堂課就做出東西
— 用 AI 對話做出網頁和遊戲，直接分享

你會學到：Prompt 四要素讓產出品質翻倍、UI/UX 五個判斷點讓作品好看好用、辨識三大 AI 平台各自的強項

帶走：2 個以上的 AI 作品（網頁 + 遊戲）

Prompt 四要素 UI/UX 判斷力 三平台比較
M1-2 封裝 AI 助手 + Gemini 圖文生成

把聊天技巧封裝成可重複使用的 AI 助手，再用 Gemini 一次產出文案和配圖

你會學到：建立自己的 GPT 和 Gem 助手、用 Gemini 圖文同時生成做行銷素材、上傳平面圖讓 AI 生成室內裝潢 3D 示意圖

帶走：2 個以上客製化 AI 助手 + 行銷圖文和裝潢示意圖

GPT / Gem 封裝 圖文同時生成 行銷應用 室內設計應用
M1-3 Google 自動化入門

拍一張菜單，AI 辨識內容並自動建立訂餐表單
— 你的第一個自動化工具

你會學到：用 Google Apps Script 讓 Google 服務自動跑、AI 幫你寫腳本你只需要說需求、遇到錯誤怎麼跟 AI 一起修

帶走：1 個實用的自動化工具（團購表單系統）

Google Apps Script OCR 數位化 自動化觸發器
M1-4 串接 AI + 做網頁 + 手機通知

做出你的第一個完整服務
— 有網頁、有 AI 智慧分析、有手機即時通知

你會學到：用 GAS 做出有公開網址的網頁、串接 Google 生態系（Sheet / Form / Gmail）、串上免費 AI API 做智慧分析、串接 Telegram Bot 即時推送

帶走：1 個完整的服務（網頁 + AI 分析 + 資料庫 + 手機通知）

GAS 網頁部署 AI API 串接 Google 生態系 Telegram Bot
S2
Stage 2 — 串 AI
跨越第二跳：從「用現成工具」到「讓 AI 幫你串流程」
M2-1 用 AI 做網頁 + GitHub Pages 上線

你的網站有短網址、有版本紀錄、你完全掌控
— 完成網頁三階段升級

你會學到：把作品部署到 GitHub Pages、認識版本控制（commit）、分辨 Public 和 Private 的差異

帶走：1 個上線的個人網站

M2-2 GitHub 安全管理 + 密碼保護

學會在 GitHub 上安全管理密碼和 API Key
— 用真實案例理解為什麼這很重要

你會學到：用 Secrets 安全存放密碼、建立自己的安全 Checklist、分辨什麼該公開什麼該保密

帶走：一套可持續使用的安全管理流程

M2-3 IDE 入門 + AI 幫你操作開發工具

進入 IDE 開發環境
— AI 幫你下指令、設定 SSH、串接 API，你只負責判斷和決策

你會學到：用 IDE 做開發、讓 AI 在終端機幫你操作 Git、設定 SSH Key、管理密碼和權限分級

帶走：一套 IDE 安全開發的標準流程

M2-4 免費 AI API 全攻略 + 期末成品

認識 5 個以上免費 AI API，了解各自的額度和限制，做出你的 Stage 2 期末作品

你會學到：選對 API 工具、了解用量限制和成本、做一個完整的 AI 應用並部署上線

帶走：1 個部署上線的 AI 應用作品

S3
Stage 3 — 造系統
跨越第三跳：從「做一個工具」到「建一套系統」
M3-1 擴展 AI 的能力
— MCP 工具 + Skills 技能

讓 AI 接上外部工具、學會你教的技能
— 從通用助手變成你的專屬系統

你會學到：用 CLI 命令列操作 AI、用 MCP 接上 GitHub 等外部工具、寫 Skills 教 AI 學會你的工作方法

帶走：你專屬的 AI 環境設定 + 3 個以上安裝的 Skills

M3-2 善用開源專案 + 安全辨識

學會找到好的開源專案、用 AI 檢查安全性、快速改造成你自己的作品

你會學到：從 GitHub 找到優質開源專案、用 AI 掃描程式碼安全性、Fork 後快速改造並部署

帶走：1 個改造自開源專案的作品 + 安全檢查清單

M3-3 系統架構思維
— 用規格書引導 AI

學會寫 spec.md 規格書
— 同一個系統，有藍圖和沒藍圖的結果天差地別

你會學到：為什麼需要先規劃再動手、怎麼寫 spec.md 跟 AI 溝通系統架構、感受有無藍圖的巨大差異

帶走：1 個 spec.md 模板 + 「先規劃再動手」的工作習慣

M3-4 自動化排程 + 雲端部署

讓你的系統 24 小時自動運轉
— 學會 GitHub Actions 和 Cloudflare Worker

你會學到：用 GitHub Actions 設定定時任務、用 Cloudflare Worker 部署 API、讓系統自動跑完後推送通知到你的手機

帶走：1 個 24/7 自動運轉的系統 + 完整的自動化藍圖
